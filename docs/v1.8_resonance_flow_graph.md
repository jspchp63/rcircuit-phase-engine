# RCIRCUIT v1.8 — Resonance Flow Graph (Skeleton)

## Purpose
The Resonance Flow Graph (RFG) describes how phase, Δsignal, coherence, and resonance energy
move through the RCIRCUIT system.  
Unlike FLOPS-based architectures that move *values*, RCIRCUIT moves *resonance states*.

This document defines the minimal public skeleton for the RFG.

---

## 1. Node Structure

Each compute node contains:

- `phase`  
- `delta_signal`  
- `coherence_local`  
- `resonance_energy`  

Nodes do **not** transmit raw values — only state changes with meaning.

---

## 2. Edge Structure

Edges represent **phase influence**, not data channels.

edge_weight = coherence_local[i] * attenuation_factor
attenuation_factor = 0.90 (default)


Influence propagates only when:

|phase_i - phase_j| <= propagation_threshold
propagation_threshold = 0.25 (default)


---

## 3. Global Flow Rules

Each update step follows:

Local Δsignal update

Local coherence recalculation

Harmonization (v1.7)

Global propagation gate (v1.6)

Resonance Flow Graph update (v1.8)


RFG updates compute:

flow_strength[i → j] = edge_weight[i → j] * (phase_i - phase_j)


---

## 4. Resonance Density (Global Metric)

Global resonance density:

R_density = Σ (coherence_local[i] * resonance_energy[i]) / N


If:
R_density >= global_activation_threshold


then the system enters:

**“Resonance Compute Mode”**  
– the RCIRCUIT equivalent of GPU parallel activation  
but with wave-based synchronization instead of FLOPS-based barriers.

---

## 5. Stability Checks (Required)

Before each propagation:

- No destructive interference (`phase_i ≠ phase_j + π`)  
- No runaway cluster oscillations  
- No coherence collapse (`coherence_local < 0.20`)  
- R_density not falling faster than decay_limit  

Default:
decay_limit = 0.05 per cycle

---

## 6. What RFG Solves

### GPUs:
- 모든 노드가 같은 clock에 묶임

### RCIRCUIT:
- 노드들이 wave처럼 정렬  
- coherence-driven flow propagation  
- unnecessary movement 제거  
- resonance density가 compute capacity가 됨
